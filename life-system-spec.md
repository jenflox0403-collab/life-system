# 라이프 시스템 대시보드 — 개발 기획서 (Claude Code 전달용)

## 프로젝트 개요

개인 인생 관리 PWA. 연간 목표부터 오늘의 10분 단위 계획까지 하나로 연결되는 "통제형 라이프 시스템".
사용자는 상황 통제 욕구가 강한 1인 콘텐츠 크리에이터(유튜브/인스타 운영, 브랜드 협업 관리).

## 기술 요구사항

- **PWA**: manifest.json + service worker, 모바일 홈 화면 추가 지원
- **반응형**: PC + 모바일 모두 최적화 (모바일 우선)
- **저장**: localStorage 기반, JSON 내보내기/가져오기 필수 (백업)
- **오프라인 동작** 가능
- 프레임워크 자유 (React 권장), 단일 배포 가능한 정적 사이트 (GitHub Pages 배포 예정)
- 다크모드 지원

## 탭 구조 (7개)

1. **오늘** (홈) — 모닝 브리핑 + 아침루틴 + 10분 타임블록 + 오늘 투두
2. **계획** — 연간 → 분기 → 월간 → 주간 캐스케이드 플래너
3. **습관** — 습관 트래커 (월별 히트맵)
4. **달력** — 광고/협업/미팅/개인 일정 관리
5. **콘텐츠** — 아이디어 → 업로드 칸반 파이프라인
6. **기록** — 일기 + 기도제목
7. **타이머** — 포모도로

---

## 1. 오늘 탭 (핵심 화면)

### 1-1. 모닝 브리핑 (자동 생성 일일 레포트)
앱을 열면 상단에 자동 표시:
- 오늘 날짜/요일, 날씨 (Open-Meteo 무료 API, 김포 기준)
- 오늘 일정 (달력에서 자동 로드)
- 오늘 투두 개수 + 최우선 항목
- 어제 습관 달성률 + 연속 달성일
- 이번 주 목표 진행률
- "오늘의 한 마디" — 사용자가 계획 탭에 적은 선언문 중 랜덤 표시

### 1-2. 아침루틴 트래커
- 순서가 있는 체크리스트 (예: 기상 → 물 한 잔 → 성경/기도 → 스트레칭 → 계획 확인)
- 각 항목에 예상 소요시간 표시, 항목 추가/수정/삭제/순서 변경 가능
- 순서대로 체크하면 진행 바 표시, 전체 완료 시 완료 애니메이션
- 완료 기록은 습관 탭 데이터와 연동 (아침루틴 자체가 습관 항목 중 하나로 집계)
- 저녁 마감 루틴도 동일 구조로 추가 (회고 작성 → 내일 타임블록 짜기 → 취침 준비)

### 1-3. 10분 단위 타임블록 플래너
- 하루 타임라인 (기본 06:00–24:00, 시작/종료 시각 설정 가능), 10분 단위 그리드
- 블록 생성: 시간 범위 드래그 또는 탭 → 제목/카테고리 입력
- 카테고리: 업무 / 콘텐츠 / 운동 / 신앙 / 개인 / 버퍼 (색상 구분)
- **버퍼 블록**을 기본 제공 — 계획이 밀렸을 때 흡수하는 여유 공간
- 투두 항목을 타임블록으로 배치 가능 (투두 → 블록 변환)
- 현재 시각 인디케이터 라인 표시
- 블록별 완료 체크 → 하루 종료 시 "계획 대비 실행률" 표시
- 어제 블록 복사 기능 (반복 일과 빠른 입력)

### 1-4. 오늘 투두 (요약)
- 오늘 날짜 투두만 표시, 우선순위(긴급/중요 조합) 색상 도트
- 전체 투두 관리는 계획 탭의 주간 화면에서

## 2. 계획 탭 (캐스케이드 플래너)

연간 → 분기 → 월간 → 주간 4단계 탭. **상위 목표에 하위 계획을 연결**하는 구조.

### 연간
- 올해 핵심 키워드 + 선언문 ("나는 ~할 것이다") 목록
- 분야별 연간 목표 (커리어/건강/재정/신앙/자기계발/관계) + 측정 지표

### 분기 (Q1–Q4)
- 연간 목표에서 파생된 분기 마일스톤
- 각 마일스톤에 상위 연간 목표 연결 표시 + 성공 기준 입력

### 월간
- 분기 마일스톤에서 파생된 월 목표
- 월 목표 달성률 진행 바

### 주간
- 이번 주 목표 + **아이젠하워 매트릭스 뷰**
- 투두 전체 목록이 여기 있고, 각 투두에 긴급/중요 속성 부여
- 매트릭스는 투두를 4분면으로 보여주는 뷰 (별도 데이터 아님)
- Q1(긴급+중요) 항목은 오늘 탭 타임블록으로 보내기 버튼
- **주간 리뷰 모드**: 일요일 저녁용 가이드 — 지난주 달성률 요약 → 잘한 점/아쉬운 점 입력 → 다음 주 목표 설정 순서로 진행되는 위저드

---

## 3. 습관 탭

- 습관 10개 (개수 조정 가능), 이름 인라인 수정
- 오늘 체크 리스트 + 항목별 연속 달성일(스트릭)
- 월별 히트맵 (달력형, 이전/다음 달 이동)
- 월간 통계: 목표 일수 대비 실행률, 가장 실패한 습관, 전체 시스템 달성률
- 연간 뷰: 12개월 달성률 그래프 (1년 회고용)

---

## 4. 달력 탭

- 월간 달력 그리드, 일정 색상 도트 표시
- 일정 유형: 광고/협찬(보라), 협업(초록), 미팅(주황), 개인(회색)
- 일정 추가: 제목, 날짜, 유형, 메모
- 협업/광고 일정에는 추가 필드: 브랜드명, 진행 상태 (문의 → 협상 → 확정 → 진행 → 정산완료)
- 날짜 클릭 시 해당일 일정 필터, 다가오는 일정 리스트 뷰
- 정산 대기 중인 건 개수 상단 뱃지 표시

---

## 5. 콘텐츠 탭 (하위 탭 3개: 파이프라인 / 위임 / 성과)

### 5-1. 파이프라인 (칸반)
- 칸반 보드: **아이디어 → 기획 → 촬영 → 편집 → 업로드 완료**
- 카드: 제목, 형태(롱폼/쇼츠/인스타), 메모, 생성일, **업로드 예정일**, 담당(나/편집자)
- 업로드 예정일이 있는 카드는 **달력 탭에 자동 표시** (콘텐츠 유형 색상)
- 빠른 아이디어 캡처: 상단 입력창에서 엔터 한 번으로 "아이디어" 열에 추가
- 카드 드래그(또는 버튼)로 단계 이동
- 업로드 완료 카드는 아카이브로 이동 (월별 업로드 개수 통계)

### 5-2. 위임 보드 (편집자 업무 관리)
- 칸반: **지시 → 진행중 → 검토 대기 → 완료**
- 카드: 업무 제목, 상세 전달사항, 마감일, 연결된 콘텐츠 카드(선택)
- 마감일 임박(D-2 이내) 카드는 빨간 뱃지, 마감 초과는 강조 표시
- "검토 대기" 개수를 콘텐츠 탭 상단 뱃지로 표시 (내가 확인할 것)
- 완료 업무 아카이브 (월별 위임 업무 개수 통계 — 편집자 성과 리뷰 시 활용)

### 5-3. 성과 기록
- 업로드 완료된 콘텐츠에 지표 기록: 조회수, 노출 클릭률(CTR), 평균 시청 지속시간, 구독 전환, 좋아요/댓글
- 인스타 콘텐츠는: 도달, 저장, 공유, 팔로우 전환
- 기록 시점 스냅샷 방식 (업로드 1주 후 / 1개월 후 등 여러 번 기록 가능)
- 정렬/필터: 조회수순, CTR순, 형태별 — "뭐가 터졌는지" 빠르게 파악
- 주간 리뷰 위저드에 "이번 주 채널 지표 회고" 단계 포함

---

## 6. 기록 탭 (일기 + 기도제목)

### 일기
- 날짜별 텍스트 + 기분 선택 (최고/좋음/보통/별로/힘듦)
- 지난 기록 리스트, 월별 기분 그래프

### 기도제목
- 기도제목 추가 (제목 + 상세 내용 + 시작일)
- **"응답됨" 체크** → 응답일 기록, 응답된 기도 아카이브 별도 보기
- 진행 중 기도제목 개수 / 응답된 기도 개수 표시

---

## 7. 타이머 탭

- 포모도로: 집중 25분 / 휴식 5분 / 긴 휴식 15분 (커스텀 가능)
- 원형 진행 링 + 오늘 완료 세션 도트
- 집중 대상 입력 → 오늘 탭 타임블록과 연동 (현재 시각 블록 자동 표시)
- 세션 완료 시 소리/진동 알림 (가능한 범위 내)

## 개발 우선순위

1. **1차**: 오늘 탭 (아침루틴 + 타임블록 + 투두) — 매일 쓰는 핵심
2. **2차**: 계획 탭 캐스케이드 + 습관 탭
3. **3차**: 콘텐츠 탭 (파이프라인 + 위임 보드) + 달력 (콘텐츠 예정일 연동 포함)
4. **4차**: 성과 기록 + 기록 탭 + 타이머 + 모닝 브리핑 날씨 API
5. **마지막**: PWA 설정, JSON 백업, 다크모드, 주간 리뷰 위저드

## 선택 사항 (구현 중 판단)

- 타이머를 별도 탭 대신 **오늘 탭 플로팅 미니 타이머**로 통합하는 방안 — 타임블록 실행 중 화면 이동 없이 사용 가능. 별도 탭과 플로팅 중 실사용 편의 기준으로 결정
- 위임 보드 카드의 전달사항을 복사 버튼으로 편집자에게 보낼 메시지 형태로 추출하는 기능

## 참고 사항

- 이전 버전(v2)에서 예산 관리 기능은 **제거** (별도 시트로 관리)
- 모든 텍스트 한국어, 폰트는 Pretendard 계열
- 디자인 톤: 미니멀, 화이트 기반, 절제된 컬러 (기능별 포인트 컬러만)
- 알림/푸시는 PWA 한계 내에서 최선 구현 (Notification API, 앱 열려있을 때 기준)
