# CLAUDE.md — 프로젝트 작업 규칙

이 파일은 Claude Code가 이 프로젝트에서 작업할 때 **반드시 준수해야 하는 규칙**입니다. 매 작업 시작 시 이 파일을 다시 확인하세요.

## 프로젝트 개요

**라이프 시스템 대시보드** — 개인 인생 관리 PWA.
상세 기획은 `life-system-spec.md` 참고. 이 기획서가 진실의 원천(source of truth)입니다.

**사용자**: 한국인 1인 콘텐츠 크리에이터 (유튜브/인스타 운영). 상황 통제 욕구가 강하며, 매일 실사용할 계획입니다. 개발 지식은 제한적이므로 코드 설명은 쉬운 한국어로.

---

## 작업 원칙 (엄격 준수)

### 1. 단계별 진행 — 절대 한 번에 다 만들지 말 것

`life-system-spec.md`의 "개발 우선순위"를 반드시 따를 것.

**각 단계 완료 전에 사용자 승인 필수.** 다음 단계로 넘어가려면 사용자가 명시적으로 "다음 단계로" 또는 이에 준하는 확인을 해야 합니다. 자의적으로 다음 단계 진행 금지.

### 2. 각 단계 시작 시 반드시 할 것

1. 이 단계에서 만들 것을 3~5줄로 요약해서 사용자에게 보여줌
2. 사용자 확인 요청 ("이대로 진행할까요?")
3. 확인 받은 후 작업 시작

### 3. 각 단계 완료 시 반드시 할 것

1. 만든 파일 목록과 각각의 역할 요약
2. **git commit 자동 실행** (commit message는 한국어로: 예 `1단계: 프로젝트 초기 세팅 완료`)
3. 사용자에게 테스트 방법 안내:
   - PC 브라우저 접속 방법 (URL)
   - **모바일 접속 방법도 반드시 안내** (로컬 IP주소 조회해서 알려줄 것)
4. 테스트 체크리스트 제공 (사용자가 뭘 확인해야 하는지 3~5개 항목)
5. "이 단계 통과하고 다음으로 갈까요, 아니면 수정할 부분 있나요?" 질문

### 4. 코드 설명 규칙

- 사용자는 코드를 잘 모릅니다. **모든 설명은 한국어로, 비유와 예시로**.
- 파일 하나 만들 때마다 "이 파일은 XX 담당" 한 줄 요약.
- 어려운 개념 등장 시 짧게 풀어서 설명 (예: "props는 부품에 전달하는 옵션 같은 거예요").
- 사용자가 "이거 왜 이래?" 물으면 절대 방어적으로 답하지 말고, 논리를 처음부터 설명.

### 5. 초반 필수 사항 (기획서 우선순위와 다름)

**1단계에서 반드시 포함할 것 (기획서 마지막 단계가 아님):**
- JSON 내보내기/가져오기 기능 (데이터 백업). 데이터 유실 방지가 최우선.
- 개발 서버가 실제로 잘 돌아가는지 확인.

### 6. 모바일 필수 확인

- **모든 화면은 모바일 반응형이어야 함.** 폭 375px (iPhone SE 기준)에서 깨지지 않아야 함.
- 각 단계 완료 시 개발 서버가 로컬 네트워크에서도 접속 가능하도록 설정 (`vite --host` 등).
- 사용자에게 모바일 접속 URL을 명확히 안내.

### 7. 파괴적 작업 금지

- **"이전에 만든 거"를 절대 임의로 수정하지 말 것.** 사용자가 명시적으로 요청한 경우만 수정.
- 대규모 리팩터링은 사용자 승인 없이 진행 금지.
- `git rm`, 폴더 삭제 등은 사용자에게 반드시 사전 확인.

### 8. 커밋 규칙

- 각 단계 완료 시 커밋 (message 한국어)
- 사용자가 "이 부분 수정해줘"라고 요청해서 수정 완료된 경우도 커밋
- 커밋 전에 무엇이 커밋되는지 사용자에게 짧게 보여줄 것

### 9. 문제 해결 규칙

- 에러 발생 시 원인 분석 → 해결 방안 → 실행 순서. 무작정 코드 수정 금지.
- 사용자가 "안 돼요"라고만 하면, 정확한 증상을 먼저 물어볼 것.
- 3번 시도해도 해결 안 되면, 사용자에게 상황 정리해서 보고하고 다른 접근 제안.

### 10. 검증 규칙

- 각 단계 완료를 선언하기 전에 **스스로 체크리스트**:
  - [ ] 기획서 요구사항 다 반영했나?
  - [ ] 개발 서버가 에러 없이 뜨나?
  - [ ] 모바일 반응형 CSS 적용했나?
  - [ ] localStorage 저장/로드가 실제로 되나?
  - [ ] 이전 단계 코드를 훼손하지 않았나?

체크리스트 하나라도 실패하면 "완료" 선언하지 말 것.

## 기술 스택 가이드

- **프레임워크**: React + Vite (빠른 개발, 쉬운 배포)
- **스타일**: Tailwind CSS (미니멀, 반응형 쉬움) 또는 CSS Modules
- **저장**: localStorage 래퍼 유틸 함수를 만들어서 사용
- **폰트**: Pretendard (한국어 최적화)
- **배포 대상**: GitHub Pages (jenflox0403-collab 계정)
- **날씨 API**: Open-Meteo (무료, 회원가입 불필요, 김포 기준)

다른 프레임워크를 제안하고 싶다면 0단계에서 사용자에게 이유와 함께 제안.

---

## 절대 하지 말 것

- ❌ 사용자 승인 없이 여러 단계 진행
- ❌ 기획서에 없는 기능 임의 추가
- ❌ 예산 관련 기능 추가 (기획서에서 명시적으로 제외됨)
- ❌ 영어로 UI 텍스트 작성 (모두 한국어)
- ❌ 어려운 코드 설명을 그대로 사용자에게 던지기
- ❌ 커밋 없이 계속 작업
- ❌ 모바일 확인 없이 완료 선언

## 시작하기

이 파일을 처음 읽은 시점이면, 다음 순서로 진행:

1. `life-system-spec.md` 읽기
2. 사용자에게 인사하고, 0단계(전체 구조 설계)를 제안
3. 폴더 구조 / 데이터 모델 / 개발 순서 / 기술 스택을 문서로 정리해서 보여줌
4. 사용자 승인 대기

> 승인해주시면 1단계부터 실제 개발을 시작하겠습니다.
