# Claude Code 프로젝트 실전 체크리스트

라이프 시스템 대시보드를 Claude Code로 만들기 위한 오너 가이드.
순서대로 따라가면 됩니다.

---

## 📦 시작 전 준비 (하루 안에 완료)

### 1. 환경 세팅
- [ ] Claude Code 설치 확인 (터미널에서 `claude` 명령어 실행되는지)
- [ ] Node.js 설치 확인 (`node -v` — v18 이상 권장)
- [ ] Git 설치 확인 (`git --version`)
- [ ] GitHub 계정 확인 (배포용, jenflox0403-collab 계정 이미 사용 중)

### 2. 프로젝트 폴더 만들기
- [ ] 작업할 폴더 생성 (예: `~/Projects/life-system/`)
- [ ] 그 폴더에 `life-system-spec.md` 넣기
- [ ] 터미널에서 그 폴더로 이동 (`cd ~/Projects/life-system/`)
- [ ] `git init` 실행 (버전 관리 시작)

### 3. 마음의 준비
- [ ] 이 프로젝트는 최소 1주일, 넉넉히 2주 걸린다고 생각하기
- [ ] 하루에 한두 단계씩만 진행 (한 번에 다 하려 하지 않기)
- [ ] 완벽한 첫 버전이 아니라 "쓸 만한 버전"을 목표로 하기

## 🎯 각 단계별 요청 템플릿

### 0단계 — 전체 구조 설계 (코드 X)
```
life-system-spec.md 파일을 읽어봐줘.
이 기획서대로 PWA를 만들 건데, 먼저 코드는 짜지 말고
아래 것들만 설계해서 보여줘:

1. 폴더 구조 (어떤 폴더에 뭐가 들어가는지)
2. 데이터 모델 (localStorage 키 구조)
3. 개발 순서 (기획서의 우선순위 참고)
4. 사용할 기술 스택 추천 (프레임워크, 라이브러리)

내가 확인하고 승인하면 그때부터 코드 작성 시작하자.
```

**✅ 오너 체크포인트:**
- [ ] 폴더 구조가 이해되는가? (모르는 부분은 질문)
- [ ] 데이터 모델이 기획서 내용을 다 담고 있는가?
- [ ] 프레임워크 선택 이유가 납득되는가?
- [ ] 이해 안 되면 "이 부분 다시 설명해줘" 라고 요청

---

### 1단계 — 프로젝트 초기 세팅
```
좋아, 이 구조로 진행하자.
프로젝트 초기 세팅만 먼저 해줘:
- package.json, 폴더 구조 만들기
- 개발 서버 실행되는지 확인
- 빈 페이지 하나 띄우기
- 데이터 저장/로드 유틸 함수 (localStorage 래퍼)
- JSON 내보내기/가져오기 기능 (백업용, 초반부터 필수)

각 파일이 어떤 역할인지 간단히 설명해줘.
완료되면 커밋해줘.
```

**✅ 오너 체크포인트:**
- [ ] 개발 서버 실행되는지 직접 확인 (`npm run dev` 또는 안내대로)
- [ ] 브라우저에서 빈 페이지가 뜨는지 확인
- [ ] JSON 내보내기 버튼 눌러서 파일 다운로드 되는지 테스트
- [ ] git 커밋 히스토리 확인 (`git log --oneline`)

---

### 2단계 — 오늘 탭 (핵심 화면)
```
기획서의 "개발 우선순위 1차"인 오늘 탭만 구현해줘.
- 모닝 브리핑 (날씨 API는 나중에, 일단 나머지만)
- 아침루틴 트래커
- 저녁 마감 루틴
- 10분 단위 타임블록 (버퍼 블록 포함)
- 오늘 투두

다른 탭은 클릭했을 때 "준비 중" 표시만 뜨게 해줘.
디자인은 기획서 톤(미니멀, 화이트 기반) 유지.
모바일에서도 잘 보이게 반응형으로.

완료되면 커밋하고, 각 컴포넌트가 어떻게 동작하는지 설명해줘.
```

**✅ 오너 체크포인트:**
- [ ] PC 브라우저에서 실제로 사용해보기 (10분 이상)
- [ ] 폰에서도 확인 (같은 와이파이면 컴퓨터 IP주소로 접속)
- [ ] 아침루틴 항목 추가/수정/삭제 다 되는가?
- [ ] 타임블록 만들기 → 카테고리 지정 → 완료 체크 흐름이 자연스러운가?
- [ ] 버퍼 블록이 시각적으로 구분되는가?
- [ ] 데이터 새로고침 후에도 유지되는가?
- [ ] 어색한 부분 메모해두기 (다음 단계 넘어가기 전에 수정 요청)

**⚠️ 피드백 요청 예시:**
```
써봤는데 몇 가지 수정하고 싶어:
1. 타임블록 만들 때 시작/종료 시각 입력이 너무 번거로워. 
   드래그로 만들 수 있게 해줘.
2. 아침루틴 완료 시 애니메이션이 너무 튀어. 더 은은하게.
3. 모바일에서 타임블록 글씨가 너무 작아.
```

---

### 3단계 — 계획 탭 + 습관 탭
```
1차 잘 동작하니까 2차로 넘어가자.
지금까지 만든 오늘 탭은 건드리지 말고,
계획 탭(연간/분기/월간/주간 캐스케이드)과 습관 탭만 추가해줘.

특히 계획 탭의 캐스케이드 연결(상위 목표 → 하위 계획 연결)이 
직관적이었으면 좋겠어.
주간 리뷰 위저드도 이 단계에서 넣어줘.

완료되면 커밋.
```

**✅ 오너 체크포인트:**
- [ ] 연간 목표 하나 만들고 → 분기 → 월간 → 주간까지 연결해보기
- [ ] 아이젠하워 매트릭스 뷰에서 투두가 잘 보이는가?
- [ ] 주간 리뷰 위저드 한 번 돌려보기
- [ ] 습관 이름 수정, 히트맵 이전/다음 달 이동 확인

---

### 4단계 — 콘텐츠 탭 + 달력
```
3차로 콘텐츠 탭(파이프라인/위임/성과 3개 하위 탭)과 
달력 탭을 만들어줘.

콘텐츠 카드에 업로드 예정일 넣으면 달력에 자동 표시되게 
연동하는 게 중요해.
위임 보드는 편집자 업무 관리용이니 마감일 강조가 잘 되어야 해.

완료되면 커밋.
```

**✅ 오너 체크포인트:**
- [ ] 아이디어 카드 하나 만들고 → 편집 단계까지 이동해보기
- [ ] 위임 카드에 마감일 넣어보고 D-day 표시 확인
- [ ] 콘텐츠 예정일이 달력에 실제로 뜨는지 확인
- [ ] 성과 기록에 가상 지표 넣고 정렬/필터 테스트

---

### 5단계 — 기록 + 타이머 + 모닝 브리핑 완성
```
4차로 마무리 기능들:
- 기록 탭 (일기 + 기도제목, 기도제목 응답 체크 포함)
- 타이머 탭
- 모닝 브리핑에 날씨 API 연동 
  (Open-Meteo, 김포 기준, 회원가입 필요 없는 무료 API)

완료되면 커밋.
```

**✅ 오너 체크포인트:**
- [ ] 일기 저장 후 지난 기록에 뜨는지
- [ ] 기도제목 등록 → "응답됨" 체크 → 아카이브 이동 확인
- [ ] 타이머 25분 완료 시 알림 뜨는지
- [ ] 다음날 아침에 모닝 브리핑이 잘 뜨는지 (실제로 다음날 확인)

---

### 6단계 — PWA 설정 + 다크모드 + 배포
```
마지막 단계야:
1. PWA 설정 (manifest.json, service worker, 아이콘)
2. 다크모드 지원 (시스템 설정 자동 감지 + 수동 토글)
3. GitHub Pages 배포 준비

배포는 내가 직접 할 테니, 배포 방법을 단계별로 설명해줘.
```

**✅ 오너 체크포인트:**
- [ ] 폰 브라우저에서 "홈 화면에 추가" 됨을 확인
- [ ] 홈 화면 아이콘 실행 시 앱처럼 열리는지
- [ ] 다크모드 토글 확인
- [ ] GitHub Pages에 배포 후 실제 URL로 접속되는지

## 🔑 매 단계 공통 규칙

### 요청할 때 항상 지킬 것
1. **"이전에 만든 거 건드리지 마"** 문구 넣기
2. **범위를 좁게** — "오늘 탭만", "달력 탭만" 식으로 명확히
3. **완료 후 커밋 요청** 잊지 않기
4. **작동 확인** 후 다음 단계로

### 이해 안 되면 바로 질문
```
이 코드가 왜 이렇게 짜여져 있는지 설명해줘.
쉽게 풀어서 알려줘.
```
코드 몰라도 됩니다. 로직 설명을 이해하는 게 중요.

### 문제 생겼을 때 대처법
```
[정확한 증상 설명]. 
예를 들어 "타임블록 만들면 화면이 하얗게 됨" 이런 식.
어떤 파일에서 문제가 났는지 찾아서 고쳐줘.
```

### 되돌리고 싶을 때
```
방금 작업 취소하고 이전 커밋으로 돌아가줘.
```

## 🚨 반드시 피해야 할 것들

- ❌ **한 번에 다 만들어달라고 요청** → 반드시 단계별로
- ❌ **첫 단계 어색한데 그냥 넘어가기** → 수정 후 진행
- ❌ **PC에서만 확인하고 폰 안 보기** → 실사용은 폰이 주력
- ❌ **커밋 안 하고 계속 작업** → 되돌리기 불가능해짐
- ❌ **"완벽하게 다시 만들어줘" 같은 모호한 요청** → 구체적으로 어디가 문제인지 지시
- ❌ **배포 후 계속 수정** → 로컬에서 안정화 후 배포

- [ ] 브라우저 캐시 지울 때 조심 (앱 데이터 날아감)
- [ ] 새 폰/기기 이동 시 JSON 가져오기로 복원

## 🎯 언제 "완성"인가

첫 배포 후 **일주일 실제로 매일 써보고** 나서:
- 매일 3번 이상 열게 되는가?
- 다른 앱(노션, 기본 캘린더 등)보다 편한가?
- 안 쓰게 되는 탭은 뭔가? → 다음 개선 방향

**완벽함 대신 "매일 여는 습관"이 진짜 목표.**

## 📞 막힐 때

- Claude Code에서 해결 안 되는 이슈 → 나(Claude)에게 스크린샷과 함께 물어봐도 됨

- 기획 자체를 바꾸고 싶을 때 → 기획서부터 수정 후 Claude Code에게 재요청
